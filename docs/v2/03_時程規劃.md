## 關鍵路徑分析

### 關鍵路徑
1. **專案初始化** → **檔案系統服務** → **檔案瀏覽 UI** → **Markdown 服務** → **預覽功能** → **MVP 完成**
2. **編輯功能** → **匯出功能** → **多平台適配** → **專案完成**

### 關鍵依賴
- 檔案系統服務必須在檔案瀏覽 UI 之前完成
- Markdown 服務必須在預覽功能之前完成
- 平台抽象層必須在平台適配之前完成

## 緩衝時間規劃

### 內建緩衝
- 每個階段預留 10-15% 緩衝時間
- 技術難題預留額外時間（PDF 匯出、Android SAF）

### 風險緩衝
- Phase 2：+2 天（檔案系統複雜度）
- Phase 3：+2 天（PDF 匯出學習曲線）
- Phase 4：+3 天（平台特定問題）

## 並行開發機會

### 可並行任務
- **T2.3.1** (MarkdownService C#) 與 **T2.3.2** (TypeScript 渲染器) 可並行
- **T3.2.1** (HTML 匯出) 與 **T3.3.1** (PDF 匯出學習) 可並行
- **T4.1.x** (macOS) 與 **T4.2.x** (Android) 可部分並行

### 並行限制
- 需要先完成基礎架構才能開始功能開發
- 需要先完成 MVP 才能開始進階功能

## 時程調整原則

### 可調整項目
- 可選功能（DOCX、ODF 匯出）可延後
- UI 增強功能可延後
- 部分測試可延後

### 不可調整項目
- MVP 核心功能（檔案瀏覽、預覽）
- 平台適配（至少 Windows + 一個其他平台）

## 進度追蹤

### 每日進度
- 每日更新任務狀態
- 記錄完成時間與實際工時

### 每週檢討
- 檢視本週完成進度
